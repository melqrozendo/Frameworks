#04 - Módulo fs(file system) para manipular arquivos em Nodejs
-é um modulo que faz parte do nucleo do nodejs
const fs = require('fs');
-acesso aos metodos assincronos por padrão:
https://nodejs.reativa.dev/0046-node-module-fs/index
mais utilizados:
fs.access()
fs.appendFile()
fs.rename()
fs.writeFile()
-os metodos do fs podem rodar de forma síncrona add Sync no nome (causa impacto no fluxo da sua aplicação):
fs.renameSync()
fs.writeSync()

*********************************
Exemplos de uso do metodo: fs.rename()

# API assíncrona é usada com uma callback (função)
sintaxe:
const fs = require('fs')
fs.rename('before.json', 'after.json', err => {
  if (err) {
    return console.error(err)
  }
  // feito
})

obs: callback é uma função, no codigo acima, esta escrita como um arrow_function que se a operação falhar deve retornar o erro;

# API síncrona pode ser usada com um bloco de try/catch para tratar erros
sintaxe: try/catch (tente/caso não)
const fs = require('fs')
try {
  fs.renameSync('before.json', 'after.json')
  // feito
} catch (err) {
  console.error(err)
}

obs: leitura do codigo, chamado o modulo fs, tente renomear de forma sincrona o arquivo 'before.json' para 'after.json', caso não, exiba o erro;
